¿Qué es el AI Feedback Loop?

  • 01
    Prompt
    Definís objetivo, constraints y contexto.
  • 02
    IA genera
    Propone código, texto o cambios.
  • 03
    Probás
    Corres, validás y encontrás fallas reales.
  • 04
    Feedback
    Ajustás el contexto y volvés a iterar.

Cuanto más rápido y fiel sea ese ciclo, mejor será el resultado.

MCPs que usamos

El feedback loop se acelera cuando la IA puede deployar, inspeccionar datos y validar la UI en tu entorno real.

  • 01
    Vercel MCP
    Deploy, preview y logs desde el agente (builds, dominios, envs).
    vercel.com/docs/agent-resources/vercel-mcp
  • 02
    Supabase MCP
    Postgres, Auth, Storage y Realtime con datos reales del proyecto.
    supabase.com/docs/guides/getting-started/mcp
  • 03
    Chrome DevTools MCP
    Automatización y debugging en Chrome (snapshots, screenshots, consola, red).
    github.com/ChromeDevTools/chrome-devtools-mcp
  • 04
    DeepWiki
    Documentación conversacional por repo: entendés código más rápido.
    deepwiki.com
  • 05
    Trustless Work (GitBook MCP)
    Docs AI-optimizadas: escrows, SDK y API en Stellar. Los sitios GitBook exponen MCP en /~gitbook/mcp para que la IA consulte documentación en tiempo real.
    docs.trustlesswork.com

/ insights

cursor-failure-insights analiza transcripciones de sesiones con la IA en multiples proyectos (22 repos, 285+ sesiones), extrae patrones de error reales y arma un slash command para aplicar esas lecciones en nuevas tareas.

  • Input: Transcripciones de chats (ej. agent-transcripts/*.jsonl) donde vos reportaste fallas y la IA corrigió.
  • Proceso: Análisis post-mortem → extracción de patrones recurrentes (compilación, tipos, sync, OAuth, paths Vercel, etc.).
  • Output: Slash command /insights con pasos y checklist para prevenir errores recurrentes.

Otras herramientas

doc.rust-lang.org/cargo/commands · github.com/StellarFoundation/scout

  • 01
    cargo check
    Verificar compilación sin generar binario (rápido).
  • 02
    cargo fmt / cargo fmt -- --check
    Formatear o verificar formato sin cambiar archivos.
  • 03
    cargo test
    Ejecutar tests.
  • 04
    cargo scout-audit (en la carpeta del contrato)
    Scout Soroban: análisis estático de vulnerabilidades.

Qué aportan al agente. Son deterministas (mismo comando → mismo resultado), ejecutables por el agente sin depender del IDE y más fiables que ReadLints para validar.
